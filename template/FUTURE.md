# FUTURE.md

Ideas worth considering later. This is a scope guard, not a commitment list.

## Admission Rules

A proposal enters scope only if it satisfies these:

1. It serves the current project goal.
2. It can be verified with a concrete check.
3. It does not add a dependency, service, or workflow without clear value.
4. It keeps agent routing simple: current truth in `README.md`, *why* in `DECISIONS.md`, procedures in `skills/`, major changes in `CHANGELOG.md`.
5. It is reversible if the project changes direction.
